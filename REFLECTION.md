--- Reflection ---
1. Git and Github is a tool that'll allow me to use the command promt to save and download versions of
   code to and from my computer and the Github site. I learned how to use some of the commands, such as
   add, commit, push, and pull.

2. Setting everying up was a bit confusing, since the instructions provided in the module left some
   things vauge, so I had to look online to get confirmation that I was doing things right. Certain
   terms were used without being told their meaning, for example, like "commit", but I think I was able
   to get it after messing around on the Github site. I was also hesitant to use add, commit, push, and
   pull on my own, without the guidence of the given instructions, but checking online sources did help
   to make sure it was all going to work as intended.

3. Developers probably use Git and Github because it helps them keep track of different versions/
   history of their code, allowing a way to see what changes have been made if something gets lost
   as a program becomes more complicated. As well, it allows a simple way for people to share their
   work with others, and to download and upload changes to someone's work through Git. Git and Github
   have become widely adopted and integrated with other platforms and/or IDEs, so it's also just an
   industry standard as well, although I assume it became the industry standard because it does its job
   efficiently.

4. With larger projects, Git and Github will allow an easy way to store version histories, so when
   something fails later down the line, we can go back and check how a program that was working
   previously was written and compare it to the changes that were made later down the line. It'll
   also be useful to make meaningful comments about what specific additions and deletions were made
   when saving code onto Github.
